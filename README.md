# Sistema PMS – Hotel Santiago

Sistema **PMS (Property Management System)** desarrollado para el **Hotel Santiago**, orientado a la gestión eficiente de reservas, habitaciones, clientes y procesos administrativos, garantizando control, seguridad y escalabilidad.

---

## 1. Descripción General

El Sistema PMS – Hotel Santiago es una plataforma diseñada para centralizar y optimizar las operaciones hoteleras, permitiendo al personal administrativo y operativo gestionar de forma ordenada y segura la información del hotel.

El sistema busca mejorar la experiencia del usuario interno, reducir errores operativos y facilitar la toma de decisiones mediante información clara y actualizada.

---

## 2. Objetivos del Proyecto

### Objetivo General
Desarrollar un sistema PMS que permita la administración integral del Hotel Santiago mediante una solución tecnológica moderna.

### Objetivos Específicos
- Gestionar reservas y disponibilidad de habitaciones
- Controlar procesos de check-in y check-out
- Administrar información de clientes
- Garantizar seguridad en el acceso a la información
- Facilitar el trabajo colaborativo del equipo de desarrollo

---

## 3. Tecnologías Utilizadas

### Backend
- **Node.js**
- **Express**
- **Socket.io**
- **JWT** (Autenticación y autorización)
- **Bcrypt** (Encriptación de contraseñas)
- **Day.js** (Gestión de fechas)

### Frontend
- Framework moderno basado en JavaScript
- Consumo de API REST
- Diseño orientado a experiencia de usuario (UX)

### Base de Datos
- En proceso de definición según requerimientos del sistema

## 6. Flujo de Trabajo con Git

### Ramas Principales
- `main` → Rama protegida, contiene versiones estables
- `backend`
- `frontend`
- `qa`

### Normas de Trabajo
- No se permite realizar push directo a `main`
- Todo cambio debe realizarse mediante **Pull Request**
- Los Pull Requests deben ser revisados antes de su aprobación

### Ejemplos de Ramas de Trabajo
- `feature/login-backend`
- `feature/reservas-frontend`
- `fix/bug-fechas`
